## 2026-05-05 - 모바일 녹음 포맷 호환성 보완

### 증상

- 일부 모바일 브라우저에서 녹음 후 생성된 오디오 파일이 정상적으로 분석되지 않을 가능성이 있었다.
- 기존 구현은 녹음 결과를 `audio/webm`으로 고정하고 있어서 iPhone, iPad, 일부 모바일 브라우저에서 호환성 문제가 생길 수 있었다.

### 재현 조건

- 모바일 브라우저에서 마이크 녹음을 사용한다.
- 브라우저가 `audio/webm` 녹음을 기본 지원하지 않거나 다른 MIME 타입을 우선 사용하는 환경이다.

### 원인 추정

- `MediaRecorder`는 브라우저마다 지원하는 MIME 타입이 다르다.
- 기존 구현은 지원 여부 확인 없이 `audio/webm`으로 Blob과 파일명을 고정하고 있었다.
- 그 결과 실제 브라우저가 생성한 포맷과 기대 포맷이 달라질 수 있었다.

### 수정 내용

- `src/app/page.tsx`에서 `MediaRecorder.isTypeSupported()`를 사용해 지원 가능한 녹음 포맷을 우선 선택하도록 변경했다.
- `audio/webm;codecs=opus`, `audio/mp4`, `audio/webm`, `audio/ogg;codecs=opus` 순서로 후보를 확인한다.
- 실제 녹음 결과의 `mimeType`을 기준으로 파일 확장자를 결정하도록 수정했다.
- 결과 파일은 브라우저가 생성한 실제 타입에 맞춰 `recorded_audio.webm`, `recorded_audio.m4a`, `recorded_audio.ogg` 등으로 저장되도록 보완했다.

### 검증 결과

- `npm run lint` 통과
- `npm run build` 통과

### 남은 리스크 / 다음 액션

- 모바일 브라우저마다 `MediaRecorder` 구현 차이가 있어 실제 기기 테스트는 계속 필요하다.
- iOS Safari 계열에서는 브라우저 버전에 따라 녹음 방식 편차가 있으므로 기기 테스트를 우선한다.

## 2026-05-05 - 모바일 STT 세션 생성 실패

### 증상

- 모바일에서 음성 분석을 시작하면 모델 다운로드는 100%까지 진행되는 것처럼 보인다.
- 하지만 실제 분석 직전에 아래 오류가 발생하며 중단된다.

```text
Can't create a session. ERROR_CODE: 1
TransposeDQWeightsForMatMulNBits
Missing required scale: model.decoder.embed_tokens.weight_merged_0_scale
```

### 재현 조건

- 모바일 브라우저에서 브라우저 로컬 Whisper 경로를 사용한다.
- 모델 준비 이후 텍스트 전사를 위한 세션을 생성하는 시점에서 오류가 발생한다.

### 원인 추정

- 프로젝트는 `@huggingface/transformers` 기반 브라우저 로컬 Whisper를 사용한다.
- WASM 경로에서 모바일 브라우저 조합에 따라 Whisper decoder 쪽 양자화 세션 생성이 불안정할 가능성이 높다.
- 오류 메시지의 `Missing required scale`, `TransposeDQWeightsForMatMulNBits`는 양자화된 가중치 복원 과정에서 나타나는 형태에 가깝다.

### 수정 내용

- `src/lib/stt/localWhisper.ts`에 모바일 및 저사양 기기 감지 로직을 추가했다.
- 모바일 경로에서는 `Xenova/whisper-tiny + WASM + 1 thread` 조합을 사용하도록 더 보수적으로 조정했다.
- 세션 생성 실패가 감지되면 더 안전한 dtype 조합으로 재시도하도록 fallback 로직을 넣었다.

### 검증 결과

- `npm run lint` 통과
- `npm run build` 통과
- 로컬 코드 기준으로 모바일 fallback 로직이 반영된 것을 확인했다.

### 남은 리스크 / 다음 액션

- `fp32` fallback은 다운로드 크기와 메모리 사용량이 늘어날 수 있다.
- 여전히 실패하면 Web Speech API fallback 또는 더 가벼운 모바일 전용 STT 경로 검토가 필요하다.

## 2026-05-05 - 모바일 STT 세션 생성 실패 2차 대응

### 증상

- 1차 대응 이후에도 모바일에서 동일한 세션 생성 오류가 반복되었다.
- 오류 메시지는 이전과 동일했다.

```text
Can't create a session. ERROR_CODE: 1
TransposeDQWeightsForMatMulNBits
Missing required scale
```

### 재현 조건

- 모바일 브라우저에서 녹음 후 분석을 시작한다.
- 모델 다운로드 이후 실제 Whisper 세션 초기화 단계에서 오류가 발생한다.

### 원인 추정

- 모바일 브라우저에서 Whisper decoder 세션 생성 자체가 불안정한 문제일 가능성이 높다.
- dtype만 바꾸는 우회만으로는 특정 모바일 브라우저 조합에서 동일한 오류를 해소하지 못했다.

### 수정 내용

- 모바일 기본 런타임을 `Xenova/whisper-tiny + WASM + fp32`로 더 보수적으로 변경했다.
- 모바일 허용 길이를 더 짧게 제한하고 chunk 길이도 줄였다.
- 오류가 반복되면 캐시된 파이프라인을 버리고 대체 경로로 다시 초기화하도록 보완했다.

### 검증 결과

- `npm run lint` 통과
- `npm run build` 통과

### 남은 리스크 / 다음 액션

- 모바일 메모리 한계로 인해 `fp32` 경로도 실패할 수 있다.
- 같은 문제가 계속되면 브라우저 로컬 Whisper만으로는 한계가 있다고 판단한다.

## 2026-05-05 - 모바일 Moonshine 분기와 STT 엔진 선택 UI 도입

### 증상

- 모바일에서는 Whisper 기반 브라우저 로컬 STT가 반복적으로 세션 생성에 실패했다.
- 반면 데스크톱에서는 Whisper 인식 품질이 만족스러워 전체 모델을 일괄 교체하기는 부담이 있었다.

### 재현 조건

- 모바일 브라우저에서 Whisper 경로를 사용할 때 오류가 반복된다.
- 데스크톱에서는 같은 흐름이 상대적으로 안정적으로 동작한다.

### 원인 추정

- 문제의 핵심은 브라우저 로컬 STT 전체가 아니라 모바일 환경에서의 Whisper ONNX 세션 생성 안정성에 있다.
- 따라서 데스크톱과 모바일에 같은 모델을 강제하기보다 환경에 따라 다른 엔진을 쓰는 편이 현실적이다.
- 사용자가 정확도 우선과 호환성 우선 중 선택할 수 있도록 UI 제공도 필요했다.

### 수정 내용

- STT 엔진 모드를 `auto`, `whisper`, `moonshine` 세 가지로 확장했다.
- `auto` 모드에서는 데스크톱은 Whisper, 모바일과 저사양 환경은 Moonshine을 우선 사용하도록 변경했다.
- `src/components/FileUploadCard.tsx`에 STT 엔진 선택 UI를 추가했다.
- 각 모드별 안내 문구도 함께 정리했다.

### 검증 결과

- `npm run lint` 통과
- `npm run build` 통과
- 코드 기준으로 자동 분기와 수동 선택 흐름이 반영된 것을 확인했다.

### 남은 리스크 / 다음 액션

- 모바일에서 Moonshine 실기기 검증이 필요하다.
- Moonshine도 모바일에서 실패하면 다음 단계는 Web Speech API fallback을 우선 검토한다.

## 2026-05-05 - 모바일 Moonshine 다운로드 이후 세션 생성 오류 지속

### 증상

- 모바일에서 `Moonshine` 모델 다운로드는 정상적으로 끝난다.
- 다운로드 직후 실제 분석이 시작되는 순간 동일한 세션 생성 오류가 다시 발생한다.
- 시크릿 모드, 새 창, 브라우저 기록 삭제 이후에도 같은 타이밍과 같은 오류 메시지가 반복된다.

```text
Can't create a session. ERROR_CODE: 1
TransposeDQWeightsForMatMulNBits
Missing required scale
```

### 재현 조건

- 배포된 서비스에 모바일 브라우저로 접속한다.
- STT 엔진을 `자동 추천` 또는 `Moonshine`으로 두고 녹음 후 분석한다.
- 모델 다운로드 100% 이후 세션 초기화 단계에서 오류가 발생한다.

### 원인 추정

- 문제의 핵심은 모델 다운로드가 아니라 ONNX Runtime 세션 생성 단계에 있다.
- 오류 메시지 형태상 양자화된 가중치를 복원하거나 초기화하는 경로에서 실패하는 가능성이 높다.
- 따라서 현재 모바일 브라우저 조합에서는 `Moonshine q8` 경로도 안정적으로 열리지 않는 것으로 추정한다.

### 수정 내용

- `src/lib/stt/localWhisper.ts`에서 `Moonshine` 런타임의 dtype을 `q8`에서 `fp32`로 변경했다.
- 이번 수정은 다운로드 용량보다 모바일 세션 호환성을 우선하는 대응이다.
- 즉 모바일 `Moonshine` 경로는 양자화 세션을 피하고 풀프리시전으로 초기화되도록 바꿨다.

### 검증 결과

- 코드 기준으로 모바일 `Moonshine` 경로가 더 이상 양자화 dtype을 사용하지 않도록 반영했다.
- 로컬 검증은 `lint`와 `build`로 다시 확인한다.

### 남은 리스크 / 다음 액션

- `fp32`로 변경하면 다운로드 크기와 메모리 사용량이 늘어날 수 있다.
- 만약 이 변경 이후에도 동일한 오류가 반복되면, 해당 모바일 브라우저에서는 브라우저 로컬 ONNX 세션 생성 자체가 어려운 것으로 판단한다.
- 그 경우 다음 우선순위는 아래와 같다.
  1. 모바일 전용 `Web Speech API` fallback 도입
  2. 모바일에서는 분석 기능을 제한하고 데스크톱 사용을 안내
  3. 모바일 전용 초경량 대체 STT 경로를 별도로 검토
